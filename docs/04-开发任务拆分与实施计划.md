# AssessV2 开发与测试基线（当前实现）

- 版本：v3.2
- 日期：2026-03-24

## 1. 目标

提供当前主流程的稳定开发基线，确保新增改动不破坏现有行为。

## 2. 基线范围

- 场次管理（含三状态流转）
- 周期/对象分组/对象维护
- 规则文件编辑与依赖检查
- 评分写入与计算对象读取
- 备份恢复与组织包
- 权限与组织范围控制

## 3. 强制校验项

每次功能改动至少满足：

1. 后端编译与测试通过：`go test ./...`
2. 前端类型检查与构建通过：`npm run build`
3. 文档同步更新（接口/字段/行为）
4. 关键写操作可追溯审计

## 4. 场次状态回归清单

1. 创建场次默认为 `preparing`
2. `preparing -> active` 成功
3. `active -> completed` 触发确认并创建完成快照
4. `completed` 下以下操作应失败：
- 更新场次
- 周期维护
- 分组维护
- 对象维护与重置
- 模块评分写入
- 规则写入
5. 从 `completed` 切回非只读状态后恢复可编辑

## 5. 权限回归清单

1. Root 可管理任意组织场次
2. Admin 仅能管理授权组织场次
3. 非授权用户修改场次状态返回权限错误

## 6. 发布前检查

1. 确认迁移脚本完整（含 `0010_assessment_session_status`）
2. 执行后端/前端构建测试
3. 复核 API 与文档一致性
4. 打包前执行一次备份演练
