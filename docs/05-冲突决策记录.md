# AssessV2 架构约束清单（当前生效）

- 版本：v3.2
- 日期：2026-03-24
- 状态：生效

## 1. 核心约束

1. 业务主实体固定为“考核场次”。
2. 场次状态固定为 `preparing / active / completed`。
3. `completed` 场次为严格只读，任何角色不可绕过。
4. 场次状态切换必须通过 `PUT /assessment/sessions/:id/status`。
5. Root/Admin 权限边界以组织范围为准，前端显隐不代表最终权限。

## 2. 数据与规则约束

1. 场次对象来源于组织主数据，不在场次内维护独立主数据。
2. 周期与对象分组均为场次内配置。
3. 对象分组编码在场次内唯一。
4. 同一场次对象目标（`targetType + targetId`）唯一。
5. 规则写操作受场次状态约束，`completed` 下只读。

## 3. 状态驱动约束

1. `preparing`：对象重置基于实时组织架构。
2. `active`：对象重置使用场次快照模式。
3. `completed`：切换时生成完成快照并锁定。

## 4. 可审计约束

以下操作必须可审计：

- 场次创建、更新、状态切换
- 周期/分组/对象替换
- 对象重置
- 规则创建与更新
- 模块评分写入
- 备份与恢复

## 5. 文档同步约束

若调整以下内容，需同步更新 `docs/`：

- 路由
- 请求/响应字段
- 状态流转规则
- 权限边界
- 数据库字段

---

本清单用于约束后续功能迭代，所有新改动需满足以上规则。
