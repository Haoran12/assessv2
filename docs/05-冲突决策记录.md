## 1. 已确认决策

1. 模块类型：**增加自定义规则模块**（`custom`）。  
说明：自定义规则可引用考核对象在当前周期所属/负责组织的各类数据；在 `YEAR_END` 可引用各季度数据并参与运算。

2. 模块唯一约束：**同类型模块允许多个实例**。  
说明：不再使用 `UNIQUE(rule_id, module_code)` 作为唯一约束。

3. `extra` 语义：**保留“权重外加减分”概念，且不占用权重、不参与权重折算**。  
说明：`extra` 与权重模块分开核算，最终总分按“加权得分 + 权重外加减分”合成。

4. 计算结果：**采用子表方案**。  
说明：新增模块级计算结果子表（如 `calculated_module_scores`），`calculated_scores` 保留聚合结果。

5. 投票数据形态：**投票不直接投分数，改为投档位**。  
说明：投票选项固定为“优/良/中/差”，按固定公式计算：  
`Σ_g ( Σ_o ( score(o) × w(g) × count(g,o) ) / total(g) )`，其中 `total(g)=Σ_o count(g,o)`。

6. 默认管理员初始化：**采用 v2.1 多角色模型**。  
说明：管理员初始化为 `users + user_roles` 两步，不使用 `users.role_id`。

7. 备份保留策略：**默认 7 天，但可配置**。  
说明：初始化配置默认 `backup.retention_days = 7`，允许系统设置修改。

8. 最高权限账号：**root**。  
说明：`username=root`，`role=root`，`password=#2026@hdwl`。

9. 系统配置表命名：**统一为 `system_settings`**。  
说明：代码与文档均按 `system_settings`，不再使用 `system_config`。

## 2. 对数据模型的直接影响

1. `score_modules`
- 新增 `module_code=custom`。
- 新增自定义规则字段（建议 `expression`, `context_scope`, `formula_version`）。
- 调整唯一约束，支持同类型多模块实例（建议新增 `module_key` 业务唯一键）。

2. `vote_records`
- 去除直接 `score` 输入，改为保存投票档位（建议字段 `grade_option`：excellent/good/average/poor）。
- 分值映射通过规则配置计算，不由投票人直接输入数值。
- 计算引擎按固定公式聚合档位票数并产出投票模块分数。

3. `extra_points`
- 业务上明确不计入权重，不参与模块权重和校验。
- 与模块关系建议弱化（可空）或独立建模（后续在 v2.2 SQL 定稿）。

4. `calculated_scores` + 子表
- `calculated_scores` 存聚合结果（加权分、最终分、计算摘要）。
- 新增 `calculated_module_scores` 存每个模块计算结果与明细。

5. 权重校验
- 仅对参与权重折算的模块（direct/vote/custom）做权重和=1.0000 校验。
- `extra` 不计入权重和。
