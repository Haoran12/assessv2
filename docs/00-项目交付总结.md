## 一、项目概述

### 1.1 项目背景

本系统为集团企业设计的年度考核管理系统，支持对集团及权属企业的各级人员和团体进行年度考核管理。系统采用本地部署方式，确保数据安全可控，便于更新和分发。系统应当在中文工作环境下运行。

### 1.2 核心需求

根据用户需求确认，系统需要满足以下核心功能：

1. **组织架构管理** - 支持集团、权属企业、部门、人员的层级管理
2. **多用户角色权限** - 超级管理员(root)、集团管理员、企业管理员、部门管理员、查看者
3. **灵活的考核规则配置** - 支持复杂表达式计算、分组投票、直接录入
4. **分组投票机制** - 同级/上级/下级分组，不同权重
5. **实时计算与排名** - 数据变更自动触发计算
6. **Excel 导入导出** - 模板导入，支持批量数据处理
7. **完善的备份机制** - 自动备份、手动备份、操作日志、数据回滚

### 1.3 技术选型确认

基于用户选择和技术评估，确定以下技术栈：

**前端**:
- Vue 3.5+ (Composition API)
- TypeScript 5.0+
- Element Plus 2.8+
- Pinia 2.2+
- vxe-table 4.6+
- ECharts 5.5+

**后端**:
- Go 1.22+
- Gin 1.10+
- GORM 1.25+
- SQLite 3.45+
- excelize 2.8+

**部署**:
- Wails 2.x 桌面应用
- Windows 10+ / macOS 12+

## 四、核心功能设计

### 4.1 考核规则配置

**分数计算公式**:
```
总分 = Σ(模块分数 × 模块权重) + 权重外加减分
```

**分数模块类型**:

1. **直接录入/导入**
   - 手动录入
   - Excel 模板导入
   - 批量录入

2. **投票计算**
   - 投票人分组：同级/上级/下级/自定义
   - 每组设置权重
   - 计算方式：
     - 简单平均
     - 加权平均

3. **自定义规则计算**
   - 支持复杂表达式
   - 可用变量：team.score, team.rank, q1.score, q2.score, q3.score, q4.score
   - 支持函数：abs(), round(), ceil(), floor(), max(), min(), if(), avg(), sum()
   - 示例：`team.score * 0.3 + if(team.rank <= 10, 5, 0)`

### 4.2 依赖关系处理

**依赖类型**:
- 个人分数依赖团体分数
- 年终分数依赖各季度分数

**处理方式**:
- 使用 DAG（有向无环图）分析依赖关系
- 拓扑排序确定计算顺序
- 并行计算无依赖项
- 缓存计算结果

### 4.3 投票机制

**投票流程**:
1. 创建投票任务
2. 系统生成投票人名单（根据规则配置）
3. 投票人登录执行投票
4. 投票完成后自动计算结果
5. 结果写入分数模块

**投票人分组**:
- 同级人员：同部门同职级
- 上级领导：直接上级或更高层级
- 下级人员：直接下级
- 自定义组：手动指定

**投票挡位**:
- 优
- 良
- 中
- 差
- 用户自定义
- 异常（不计分）
- 每个挡位可设置对应分数

### 4.4 备份与恢复

**自动备份**:
- 每日中午12点自动备份
- 保留最近 7 天
- gzip 压缩存储

**手动备份**:
- 随时创建备份
- 可添加备份说明
- 关键节点备份（季度结束、年终结束）

**数据恢复**:
- 完整数据库恢复
- 单条记录回滚（基于操作日志）
- 恢复前自动创建备份

## 八、技术风险与应对

### 8.1 主要风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 复杂表达式解析性能 | 高 | 使用成熟库（govaluate/expr），提前性能测试 |
| 依赖关系计算复杂 | 高 | DAG 算法，充分测试边界情况 |
| SQLite 并发限制 | 中 | 事务和锁机制，考虑读写分离 |
| Wails 打包兼容性 | 中 | 提前在目标平台测试 |
| Excel 格式多样性 | 中 | 严格模板，完善验证 |

### 8.2 技术难点

1. **复杂表达式计算引擎** - 使用 govaluate 或 expr 库
2. **依赖关系分析** - 拓扑排序算法
3. **实时计算通知** - WebSocket 推送
4. **大数据量表格** - 虚拟滚动、分页加载
