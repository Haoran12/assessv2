# AssessV2 系统实现总览

- 更新日期：2026-03-24
- 文档版本：v3.2
- 口径：当前代码与迁移结果

## 1. 交付结论

系统当前实现稳定运行于“场次中心”模型，前后端行为已统一。

## 2. 当前可用模块

后端路由组：

- `/api/auth`
- `/api/org`
- `/api/assessment`
- `/api/rules`
- `/api/backup`
- `/api/system`

前端页面：

- 考核主页
- 考核管理
- 规则管理
- 组织架构
- 用户管理（Root）
- 备份恢复
- 审计日志
- 系统设置（Root）

## 3. 场次三状态实现

状态枚举：

- `preparing`
- `active`
- `completed`

状态行为：

1. `preparing`
- 允许编辑场次基础信息、周期、对象分组、对象、评分与规则。
- 对象初始化/重置基于当前组织架构。

2. `active`
- 允许编辑。
- 对象重置优先使用场次快照模式，避免受组织结构变化影响。

3. `completed`
- 切换时创建并记录场次唯一完成快照。
- 场次数据与规则只读（包含 Root）。
- 仅在“考核管理”将状态切回非 `completed` 后，才允许继续修改。

## 4. 权限边界（状态变更）

- 仅 Root 与场次所属组织 Admin 可调整场次状态。
- 状态变更接口：`PUT /api/assessment/sessions/:id/status`

## 5. 规则与评分

- 规则通过 `rule_files` 管理。
- 规则按场次上下文编辑并支持依赖检查。
- `completed` 场次下规则编辑、评分录入均被服务层拒绝。

## 6. 备份能力

- 支持整库备份恢复（`/api/backup/records`）。
- 支持组织级逻辑备份包（`/api/backup/org-packages`）。
