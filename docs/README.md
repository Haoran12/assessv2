# AssessV2 文档总览（2026-03-24）

本目录文档仅描述当前已实现能力，口径与代码一致。

## 核心口径

- 业务主实体为“考核场次（assessment session）”。
- 场次状态为 `preparing / active / completed`。
- 场次处于 `completed` 时，场次数据与规则进入只读，任何角色（含 Root）都不能在业务入口修改。
- 场次切换到 `completed` 时会生成并记录该场次唯一完成快照。
- 场次对象初始化与重置逻辑：
  - `preparing`：基于当前组织架构实时生成。
  - `active`：使用场次快照模式。
  - `completed`：只读，不允许重置或改写。
