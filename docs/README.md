# AssessV2 文档总览（2026-03-18）

本目录文档已按当前代码与数据库实现重写，核心口径如下：

- 架构以“考核场次（assessment session）”为中心，不再以年度模板驱动业务。
- 顶部全局上下文为：`考核场次 / 周期 / 考核对象类型`。
- Root 与 Admin 基于组织范围管理场次与规则，不再与考核对象做静态绑定。
- 每个场次数据独立，规则文件采用“绑定即拷贝”，避免跨组织互相影响。

## 当前实现边界说明

- 生效后端路由见 `backend/internal/api/router/router.go`。
- 生效业务迁移以 `backend/migrations/business/0001 + 0002`、`backend/migrations/accounts/1001 + 1002` 为准。
- 旧的“按年度规则模板、投票任务、对象链接”等数据模型已被业务迁移移除。
- 仓库内仍保留部分历史模型/服务文件用于过渡，不代表当前主流程。
