## 项目概述

本系统是为集团企业设计的年度考核管理系统，支持本地部署，提供组织架构管理、考核规则配置、分数录入、投票管理、自动计算和报表统计等功能。

### 核心特性

- ✅ **本地部署** - Tauri 桌面应用，数据安全可控
- ✅ **多用户协作** - 支持多角色权限管理
- ✅ **灵活配置** - 可视化规则配置，支持复杂表达式计算
- ✅ **分组投票** - 支持多组投票，不同权重
- ✅ **实时计算** - 数据变更自动触发计算和排名
- ✅ **完善备份** - 自动备份、手动备份、操作日志、数据回滚

### 技术栈

**前端**: Vue 3 + TypeScript + Element Plus + Pinia + vxe-table + ECharts  
**后端**: Go + Gin + GORM + SQLite  
**桌面**: Tauri 2.0  
**平台**: Windows 10+ / macOS 12+

## 核心功能说明

### 1. 组织架构

```
集团
├── 集团高层领导
├── 集团部门
│   ├── 管理人员
│   └── 一般人员
└── 权属企业
    ├── 企业高层领导
    └── 企业部门
        ├── 管理人员
        └── 一般人员
```

### 2. 考核周期

每个年度包含 5 个考核周期：
- Q1（第一季度）
- Q2（第二季度）
- Q3（第三季度）
- Q4（第四季度）
- YEAR_END（年终考核）

### 3. 分数计算公式

```
总分 = Σ(模块分数 × 模块权重) + 权重外加减分
```

**分数模块类型**：
1. **直接录入** - 手动录入或 Excel 导入
2. **投票计算** - 分组投票，支持多种计算方式
3. **自定义规则** - 基于表达式的复杂计算

### 4. 投票机制

- 投票人分组：同级/上级/下级/自定义
- 每组设置权重
- 计算方式：简单平均/加权平均/去极值平均

### 5. 依赖关系

- 个人分数依赖团体分数
- 年终分数依赖各季度分数
- 使用 DAG 算法处理依赖关系

## 数据精度

- **后端存储**: 6 位小数（DECIMAL(10, 6)）
- **后端计算**: float64，保持 6 位小数精度
- **前端展示**: 2 位小数，四舍五入
- **排名计算**: 使用完整精度（6 位小数）

## 约束与限制

- 最大权属企业数: 50
- 最大部门数: 200
- 最大人员数: 1000
- 最大考核对象数: 200（单年度）
- 最大分数模块数: 10（单规则）
- SQLite 数据库文件大小建议 < 2GB
