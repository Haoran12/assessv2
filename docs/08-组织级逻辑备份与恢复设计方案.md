# AssessV2 备份与恢复说明（当前实现）

- 版本：v3.2
- 日期：2026-03-24

## 1. 支持的备份类型

## 1.1 全量备份

接口：`/api/backup/records`

- 创建、列表、下载、删除、恢复
- 用于整库级灾难恢复

## 1.2 组织级逻辑备份包

接口：`/api/backup/org-packages`

- 创建组织包
- 查询组织包
- 下载组织包
- 恢复组织包

## 2. 权限模型

- 全量备份：`backup:view` / `backup:update`
- 组织包：`backup:org:view` / `backup:org:update`
- 组织范围校验由后端执行

## 3. 路由清单

- `GET /api/backup/records`
- `POST /api/backup/records`
- `GET /api/backup/records/:id/download`
- `DELETE /api/backup/records/:id`
- `POST /api/backup/records/:id/restore`
- `GET /api/backup/org-packages`
- `POST /api/backup/org-packages`
- `GET /api/backup/org-packages/:id/download`
- `POST /api/backup/org-packages/:id/restore`

## 4. 恢复安全控制

- 恢复接口要求确认参数。
- 恢复过程记录审计日志。
- 权限与组织范围不满足时直接拒绝。

## 5. 运行建议

1. 变更前先执行手工全量备份。
2. 组织级数据迁移优先使用组织包。
3. 定期抽样验证“下载 + 恢复”链路。
