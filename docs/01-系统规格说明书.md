# AssessV2 系统规格说明书（接口对接版）

- 版本：v3.1
- 日期：2026-03-18
- 适用范围：当前代码实现（后端 Gin API + 前端对接）

## 1. 业务总览

系统以“考核场次（assessment session）”为核心实体，不再以“年度模板”驱动主流程。

核心业务闭环：

1. Root/Admin 创建并管理考核场次
2. 在场次内配置周期与对象分组
3. 维护场次对象（默认生成 + 手动增删）
4. 在规则管理中按“场次/周期/对象分组”绑定规则
5. 绑定时自动创建规则拷贝，按组织隔离后续编辑

## 2. 全局对接规范

## 2.1 基础信息

- Base Path：`/api`
- 认证：`Authorization: Bearer <token>`
- Content-Type：`application/json`

## 2.2 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "traceId": "optional"
}
```

## 2.3 常用业务码

| code | 含义 |
| --- | --- |
| 200 | 成功 |
| 40001 | 请求体非法 |
| 40002 | 参数非法 |
| 40003 | 业务规则不满足 |
| 40100 | 未认证 |
| 40101 | 用户名或密码错误 |
| 40301 | 权限不足 |
| 40302 | 组织范围不足 |
| 40401 | 资源不存在 |
| 50001 | 服务内部错误 |

## 4. 认证接口

## 4.1 登录

- 方法：`POST /api/auth/login`
- 权限：公开接口

请求字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

请求示例：

```json
{
  "username": "root",
  "password": "#AssessV2@Init"
}
```

响应 `data` 字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| token | string | JWT Token |
| tokenType | string | 固定 `Bearer` |
| expiresIn | number | 过期秒数 |
| mustChangePassword | boolean | 是否强制改密 |
| user | object | 用户信息 |

`user` 子字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 用户 ID |
| username | string | 用户名 |
| role | string | 主角色 |
| roles | string[] | 角色列表 |
| permissions | string[] | 权限点列表 |
| organizations | object[] | 组织范围 |
| permissionBindings | object[] | 权限绑定明细 |

响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "<jwt>",
    "tokenType": "Bearer",
    "expiresIn": 86400,
    "mustChangePassword": true,
    "user": {
      "id": 1,
      "username": "root",
      "role": "root",
      "roles": ["root"],
      "permissions": ["*"],
      "organizations": [],
      "permissionBindings": []
    }
  }
}
```

## 4.2 修改密码

- 方法：`POST /api/auth/change-password`
- 权限：已登录

请求字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| oldPassword | string | 是 | 原密码 |
| newPassword | string | 是 | 新密码（至少 8 位） |

响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "changed": true
  }
}
```

## 4.3 登出

- 方法：`POST /api/auth/logout`
- 权限：已登录

响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "loggedOut": true
  }
}
```

响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "items": [
      { "id": 21, "assessmentId": 12, "periodCode": "Q1", "periodName": "第一季度", "sortOrder": 1 }
```

## 6. 规则管理接口

## 6.1 规则文件列表

- 方法：`GET /api/rules/files?assessmentId=:id&includeHidden=false`
- 权限：`rule:view`

返回字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 规则文件 ID |
| assessmentId | number | 场次 ID |
| ruleName | string | 规则名 |
| description | string | 描述 |
| contentJson | string | 规则 JSON 字符串 |
| filePath | string | 物理文件路径 |
| isCopy | boolean | 是否拷贝规则 |
| sourceRuleId | number | 源规则 ID（拷贝规则有） |
| ownerOrgId | number | 拷贝归属组织（拷贝规则有） |
| hiddenByCurrentOrg | boolean | 当前组织是否隐藏该基础规则 |
| canEdit | boolean | 当前用户可编辑 |
| canDelete | boolean | 当前用户可删除 |

## 6.2 创建规则文件（基础规则）

- 方法：`POST /api/rules/files`
- 权限：`rule:update`（服务层限制 Root）

请求字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| assessmentId | number | 是 | 场次 ID |
| ruleName | string | 是 | 规则名称 |
| description | string | 否 | 描述 |
| contentJson | string | 否 | 规则 JSON，空则使用默认模板 |

请求示例：

```json
{
  "assessmentId": 12,
  "ruleName": "基础规则-经营指标",
  "description": "Root 创建",
  "contentJson": "{\n  \"version\": 2,\n  \"scoreModules\": [],\n  \"gradeRules\": []\n}"
}
```

## 6.3 更新规则文件（仅拷贝规则）

- 方法：`PUT /api/rules/files/:id`
- 权限：`rule:update`
- 约束：
  - 基础规则不可编辑
  - Admin 仅可编辑自己组织拷贝

请求字段同创建。

## 6.4 删除规则文件

- 方法：`DELETE /api/rules/files/:id`
- 权限：`rule:update`

约束：

- 基础规则：仅 Root 可删
- 拷贝规则：Root 可删；Admin 仅可删自己组织拷贝

成功响应：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "deleted": true
  }
}
```

## 6.5 隐藏/恢复基础规则

- 隐藏：`POST /api/rules/files/:id/hide`
- 恢复：`DELETE /api/rules/files/:id/hide`
- 权限：`rule:view`
- 约束：仅 Admin，且只能对基础规则操作

## 7. 组织架构接口（/api/org）

- 鉴权：均需 JWT + `RequireOrgScope` 中间件。
- 备注：写接口还会校验组织写入范围；Root 不受组织范围限制。

## 7.1 组织树

- 方法：`GET /api/org/tree?includeInactive=false`
- 权限：`org:view`

查询字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| includeInactive | boolean | 否 | `true` 时包含停用组织/部门/人员，默认 `false` |

响应 `items[]` 字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 节点 ID |
| nodeType | string | `organization` / `department` / `employee` |
| name | string | 节点名称 |
| status | string | 状态，常见 `active` / `inactive` |
| organizationId | number | 组织 ID（组织/部门/人员节点可有） |
| departmentId | number | 部门 ID（部门/人员节点可有） |
| parentId | number | 父节点 ID（组织 parent 或部门 parent） |
| sortOrder | number | 排序 |
| children | array | 子节点 |

响应示例（节选）：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "items": [
      {
        "id": 7,
        "nodeType": "organization",
        "name": "华东公司",
        "status": "active",
        "organizationId": 7,
        "sortOrder": 10,
        "children": [
          {
            "id": 101,
            "nodeType": "department",
            "name": "市场部",
            "status": "active",
            "organizationId": 7,
            "departmentId": 101,
            "sortOrder": 100
          }
        ]
      }
    ]
  }
}
```

## 7.2 组织列表与增删改

- 列表：`GET /api/org/organizations?status=active&keyword=华东`
- 创建：`POST /api/org/organizations`
- 更新：`PUT /api/org/organizations/:id`
- 删除：`DELETE /api/org/organizations/:id`
- 权限：列表 `org:view`，其余 `org:update`

创建/更新请求字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| orgName | string | 是 | 组织名称 |
| orgType | string | 是 | `group` / `company` |
| parentId | number | 否 | 父组织 ID，Admin 创建时必须在授权范围内 |
| leaderId | number | 否 | 负责人员工 ID |
| sortOrder | number | 否 | 排序 |
| status | string | 否 | `active` / `inactive`，默认 `active` |

列表响应 `items[]` / 创建更新响应 `data` 字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 组织 ID |
| orgName | string | 组织名称 |
| orgType | string | 组织类型 |
| parentId | number | 父组织 ID |
| leaderId | number | 负责人 ID |
| sortOrder | number | 排序 |
| status | string | 状态 |
| createdBy | number | 创建人 ID |
| createdAt | number | 创建时间戳 |
| updatedBy | number | 更新人 ID |
| updatedAt | number | 更新时间戳 |

创建请求示例：

```json
{
  "orgName": "华东供应链公司",
  "orgType": "company",
  "parentId": 7,
  "leaderId": 3001,
  "sortOrder": 120,
  "status": "active"
}
```

删除成功响应：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "deleted": true
  }
}
```
