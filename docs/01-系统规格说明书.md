## 1. 项目概述

### 1.1 项目背景

本系统为集团企业设计的年度考核管理系统，用于管理集团及权属企业的组织架构、考核规则配置、分数录入、投票管理和结果计算。

### 1.2 项目目标

- 提供清晰的组织架构管理
- 支持灵活的考核规则配置
- 实现自动化的分数计算和排名
- 支持多用户协作和权限管理
- 便于本地部署、更新和数据备份

### 1.3 技术选型

#### 前端技术栈
- **框架**: Vue 3.5+ (Composition API)
- **语言**: TypeScript 5.0+
- **UI 组件库**: Element Plus 2.8+
- **状态管理**: Pinia 2.2+
- **表格组件**: vxe-table 4.6+
- **图表库**: ECharts 5.5+
- **构建工具**: Vite 5.0+
- **表达式解析**: expr-eval
- **Excel 处理**: SheetJS (xlsx)

#### 后端技术栈
- **语言**: Go 1.22+
- **Web 框架**: Gin 1.10+
- **ORM**: GORM 1.25+
- **数据库**: SQLite 3.45+
- **Excel 处理**: excelize 2.8+
- **定时任务**: robfig/cron 3.0+
- **配置管理**: viper 1.18+
- **日志**: zap 1.27+

#### 桌面应用
- **框架**: Tauri 2.0+
- **平台**: Windows 10+ / macOS 12+
- **更新**: Tauri 内置更新器

### 1.4 部署方式

- **主要方式**: Tauri 桌面应用（单机运行，自带数据库）
- **数据存储**: SQLite 数据库文件（位于应用数据目录）
- **更新机制**: Tauri 内置更新器，支持增量更新

## 2. 功能需求

### 2.1 用户认证与权限管理

#### 2.1.1 用户角色定义

| 角色 | 权限范围 | 典型操作 |
|------|---------|---------|
| **超级管理员** | 全部功能 | 系统配置、用户管理、所有数据的增删改查 |
| **集团管理员** | 集团范围 | 管理集团部门和人员、配置考核规则、查看所有数据 |
| **企业管理员** | 所属企业范围 | 管理本企业部门和人员、录入分数、查看本企业数据 |
| **部门管理员** | 所属部门范围 | 录入本部门人员分数、查看本部门数据 |
| **查看者** | 只读权限 | 查看被授权范围的考核数据和报表 |

#### 2.1.2 认证机制
- 用户名 + 密码登录
- 密码采用 bcrypt 加密存储（cost=10）
- 登录会话管理（JWT Token，有效期 24 小时）
- 支持修改密码和重置密码（超级管理员操作）
- 首次登录强制修改密码

#### 2.1.3 权限控制
- 基于角色的访问控制（RBAC）
- 数据范围权限（只能访问所属组织的数据）
- 操作权限（增删改查的细粒度控制）
- API 级别的权限校验

### 2.2 组织架构管理

#### 2.2.1 组织层级结构

```
集团（固定唯一）
├── 集团高层领导
├── 集团部门 1
│   ├── 部门管理人员
│   └── 一般人员
├── 集团部门 2
├── 权属企业 1
│   ├── 企业高层领导
│   ├── 企业部门 1-1
│   │   ├── 部门管理人员
│   │   └── 部门一般人员
│   └── 企业部门 1-2
└── 权属企业 2
```

#### 2.2.2 组织实体定义

**集团（固定唯一）**
- 集团名称
- 集团代码
- 创建时间

**权属企业**
- 企业名称
- 企业代码
- 所属组织（集团或上级企业）
- 企业负责人
- 排序号
- 状态（启用/停用）

**部门**
- 部门名称
- 所属组织（集团或权属企业）
- 部门负责人
- 排序号
- 状态（启用/停用）

**人员**
- 姓名
- 所属组织（集团部门或权属企业领导班子或企业部门）
- 职级（集团高层领导/企业高层领导/正职管理人员/副职管理人员/一般人员）
- 入职日期
- 状态（在职/离职）

#### 2.2.3 功能要求
- 支持组织架构的增删改查
- 支持人员在部门间调动（保留历史记录）
- 支持批量导入组织架构（Excel 模板）
- 支持组织架构树形展示和搜索
- 停用的组织不参与考核，但保留历史数据

### 2.3 考核周期管理

#### 2.3.1 年度管理
- 手动创建新年度
- 创建时可选择从上一年度复制：
  - 组织架构
  - 考核规则配置
  - 分数模块定义
- 年度状态：筹备中/进行中/已结束
- 年度间数据完全隔离

#### 2.3.2 考核周期
每个年度包含 5 个考核周期：
- 第一季度（Q1）
- 第二季度（Q2）
- 第三季度（Q3）
- 第四季度（Q4）
- 年终考核（YEAR_END）

周期状态：未开始/进行中/已结束/已锁定

### 2.4 考核对象分类

#### 2.4.1 团体考核对象
- 集团部门
- 权属企业
- 权属企业部门

#### 2.4.2 个人考核对象
按职级分类：
- 集团高层领导
- 权属企业高层领导
- 管理人员（集团部门管理人员、企业部门管理人员）
- 一般人员（集团部门一般人员、企业部门一般人员）

#### 2.4.3 考核对象属性
- 考核对象类型（团体/个人）
- 考核对象级别（对应上述分类）
- 所属关系（人员属于团体，部门属于集团或企业）
- 参与考核的年度和周期

### 2.5 考核规则配置

#### 2.5.1 规则配置维度
规则按以下维度配置：
- **年度**: 2024、2025...
- **周期**: Q1/Q2/Q3/Q4/YEAR_END
- **对象类型**: 团体/个人
- **对象级别**: 集团部门/权属企业/企业部门/集团高层/企业高层/正职管理人员/副职管理人员/一般人员

同一年度的 Q1-Q4 默认使用相同规则，年终单独配置。

#### 2.5.2 分数计算公式

```
总分 = Σ(模块分数 × 模块权重) + 权重外加减分
```

**约束条件**：
- 参与权重折算的模块权重之和 = 1.0（权重外加减分不占用权重）
- 权重外加减分范围：-20 ~ +20 分
- 最终总分范围：0 ~ 120 分

#### 2.5.3 分数模块类型

**类型 1: 直接录入/导入**
- 管理员手动录入分数
- 或通过 Excel 批量导入
- 设置：模块名称、权重、满分值

**类型 2: 投票计算**
- 由指定人员组投票（选择档位：优/良/中/差）
- 投票人分组：
  - 同级人员
  - 上级领导
  - 下级人员
  - 服务对象
  - 自定义组
- 每组设置权重
- 计算方式（固定公式）：
  - 档位映射：优/良/中/差按规则映射为分值
  - 设 `score(o)` 为档位 `o` 的分值，`w(g)` 为权重组 `g` 的投票人权重，`count(g,o)` 为权重组 `g` 中选择档位 `o` 的票数，`total(g)` 为权重组 `g` 总票数
  - 投票模块得分：
    ```
    Σ_g ( Σ_o ( score(o) × w(g) × count(g,o) ) / total(g) )
    ```
  - 其中：
    ```
    total(g) = Σ_o count(g,o)
    ```
- 设置：模块名称、权重、投票组配置、档位映射规则

**类型 3: 自定义规则计算**
- 基于组织上下文和周期上下文数据计算
- 支持复杂表达式
- 可用变量：
  - `team.score`: 所属团体的总分
  - `team.rank`: 所属团体的排名
  - `q1.score`, `q2.score`, `q3.score`, `q4.score`: 各季度总分
  - `org.*`: 考核对象在当前周期所属/负责组织的业务数据
  - `module_xxx`: 其他模块的分数
  - `extra_points`: 权重外加减分
- 年终周期（`YEAR_END`）可引用各季度数据并进行综合运算
- 支持函数：
  - 数学函数：`abs()`, `round()`, `ceil()`, `floor()`, `max()`, `min()`
  - 条件函数：`if(condition, true_value, false_value)`
  - 统计函数：`avg()`, `sum()`
- 示例表达式：
  ```
  team.score * 0.3 + if(team.rank <= 10, 5, 0)
  (q1.score + q2.score + q3.score + q4.score) / 4 * 0.8
  ```
- 设置：模块名称、权重、计算表达式

#### 2.5.4 规则配置界面要求
- 可视化配置界面
- 支持添加/删除/排序模块
- 实时显示权重总和（必须等于 1.0）
- 表达式编辑器（语法高亮、自动补全、实时验证）
- 支持规则模板保存和复用
- 支持从其他年度/周期复制规则

### 2.6 分数数据管理

#### 2.6.1 直接录入分数
- 按考核对象、周期录入
- 支持单条录入和批量录入
- 支持 Excel 模板导入
- 录入时校验：
  - 分数范围（0 ~ 模块满分）
  - 必填项检查
  - 重复录入提示
- 记录录入人和录入时间
- 支持修改和删除（记录操作日志）

#### 2.6.2 投票数据管理

**投票发起**
- 选择考核对象和周期
- 选择投票模块
- 系统自动生成投票任务
- 确定投票人名单（根据规则配置）

**投票执行**
- 投票人登录后看到待投票任务
- 对每个被考核对象选择档位（优/良/中/差）
- 可选填写评价意见
- 支持保存草稿和提交
- 提交后不可修改（特殊情况管理员可重置）

**投票统计**
- 实时显示投票进度
- 投票完成后自动计算模块分数
- 支持查看投票明细（按权限）
- 支持导出投票结果

#### 2.6.3 权重外加减分
- 单独录入，不属于任何模块
- 支持正负分数（-20 ~ +20）
- 必须填写加减分原因
- 需要审批流程（可选）

### 2.7 分数计算与排名

#### 2.7.1 计算触发机制
- 数据变更时自动触发
- 支持手动触发重算：
  - 单个考核对象
  - 整个部门/企业
  - 全部考核对象

#### 2.7.2 依赖关系处理
- 个人分数依赖团体分数
- 年终分数依赖各季度分数
- 使用 DAG（有向无环图）分析依赖
- 按依赖顺序计算，支持并行计算无依赖项

#### 2.7.3 计算结果
- 各模块分数
- 权重外加减分
- 总分（保留 6 位小数存储，前端显示 2 位小数）
- 排名（同类型同级别内排名）
- 计算时间戳

#### 2.7.4 排名规则
- 按总分降序排名
- 总分相同时按模块分数优先级排名（可配置）
- 支持分组排名：
  - 集团部门内排名
  - 权属企业内排名
  - 全集团排名

### 2.8 数据导入导出

#### 2.8.1 Excel 模板导入
提供标准模板：
- 组织架构导入模板
- 人员信息导入模板
- 直接录入分数导入模板

模板要求：
- 固定表头（中文列名）
- 必填字段标注
- 数据格式说明

导入流程：
1. 下载模板
2. 填写数据

5. 显示校验结果（成功/失败/警告）
6. 确认导入
7. 显示导入结果统计

#### 2.8.2 Excel 导出
支持导出：
- 考核结果汇总表
- 考核对象明细表
- 投票统计表

导出格式：
- .xlsx 格式
- 包含表头和数据
- 支持多 Sheet（按部门/企业分组）
- 自动列宽和样式

### 2.9 报表与统计

## 4. 数据精度说明

### 4.1 分数存储与计算
- **后端存储**: DECIMAL(10, 6) - 保留 6 位小数
- **后端计算**: 使用 float64，保持 6 位小数精度
- **前端展示**: 保留 2 位小数，四舍五入
- **排名计算**: 使用完整精度（6 位小数）比较

### 4.2 权重处理
- 权重存储: DECIMAL(5, 4) - 如 0.3000
- 权重总和校验: 精确等于 1.0000
- 权重计算: 使用完整精度

## 5. 约束与限制

### 5.1 数据量限制
- 最大权属企业数: 20
- 最大部门数: 200
- 最大人员数: 5000
- 最大考核对象数: 500（单年度）
- 最大分数模块数: 10（单规则）

### 5.2 业务规则约束
- 已锁定的周期不可修改数据
- 已结束的年度不可删除
- 在职人员不可删除，只能停用
- 投票提交后不可修改（管理员可重置）

### 5.3 技术限制
- SQLite 数据库文件大小建议 < 2GB
- 单次 Excel 导入行数 < 5000
