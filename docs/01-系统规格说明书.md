# AssessV2 系统规格说明书（当前实现）

- 版本：v3.2
- 日期：2026-03-24
- 适用范围：当前代码实现（Gin API + Vue 前端）

## 1. 业务总览

系统以“考核场次”为核心，主流程：

1. 创建场次并维护场次配置
2. 维护周期与对象分组
3. 维护考核对象与评分
4. 编辑规则文件并按上下文生效
5. 按状态进行场次治理（preparing/active/completed）

## 2. 全局约定

- Base Path：`/api`
- 认证：`Authorization: Bearer <token>`
- 响应：`{ code, message, data, traceId? }`

## 5. 场次状态接口

### 5.1 更新状态

- 方法：`PUT /api/assessment/sessions/:id/status`
- 权限：`assessment:update` + 组织范围校验

请求体：

```json
{
  "status": "preparing"
}
```

可选值：`preparing`、`active`、`completed`

### 5.2 业务规则

1. 仅 Root 与场次所属组织 Admin 可调整状态。
2. 切换到 `completed`：创建并记录场次唯一完成快照。
3. `completed` 场次写操作统一禁止：
- 场次信息修改
- 周期/对象分组/对象维护
- 对象重置
- 模块评分写入
- 规则写入
4. 切回 `preparing` 或 `active` 后，写操作恢复。

## 6. 对象重置规则

接口：`POST /api/assessment/sessions/:id/objects/reset-default`

- `preparing`：基于当前组织架构重建默认对象。
- `active`：基于场次快照模式恢复。
- `completed`：禁止执行（返回业务规则错误）。

## 7. 常见错误语义

- `40001/40002`：请求结构或参数错误
- `40003`：业务规则失败（如 `completed` 场次只读）
- `40301/40302`：权限不足或组织范围不足
- `40401`：资源不存在

## 8. 对接建议

1. 所有写操作先读取场次 `status`，前端提前禁用无效按钮。
2. 状态改为 `completed` 时前端必须二次确认。
3. 对 `40003` 做业务提示，不当作系统异常。
