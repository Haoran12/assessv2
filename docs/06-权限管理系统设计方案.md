# AssessV2 权限管理说明（当前实现）

- 版本：v3.2
- 日期：2026-03-24

## 1. 设计原则

1. 功能权限与组织范围分离。
2. 后端是唯一权限裁决层。
3. 场次状态参与写权限判定。

## 2. 身份与范围

JWT Claims 关键字段：

- `roles`
- `permissions`
- `orgScopes`
- `permissionBindings`

角色：

- `root`
- `assessment_admin`
- `leader`
- `staff`

## 3. 权限判定链路

1. `RequireJWT`
2. `RequirePermission`
3. 服务层组织范围校验
4. 服务层状态写保护（`completed`）

## 4. 场次权限矩阵

## 4.1 查看

- Root：任意组织
- Admin：授权组织范围

## 4.2 修改（基础信息/周期/分组/对象/评分/规则）

- Root：任意组织，但受 `completed` 只读限制
- Admin：授权组织，且受 `completed` 只读限制

## 4.3 状态切换

接口：`PUT /api/assessment/sessions/:id/status`

- Root：可改任意场次状态
- Admin：仅可改授权组织场次状态
- 非以上角色：不可改

## 5. completed 只读策略

当场次状态为 `completed`：

- 场次配置写操作全部拒绝
- 评分写操作拒绝
- 规则写操作拒绝
- 即使 Root 也不例外

## 6. 前后端协作约定

- 前端基于 `canEdit`、`status` 做交互限制。
- 后端始终再次校验，拒绝越权与只读写入。

## 7. 审计要求

需记录审计的关键操作：

- 场次状态变更（含 from/to）
- `completed` 快照创建
- 场次核心配置写操作
- 评分与规则写操作

---

本文件为当前权限行为基准，权限点或边界变化需同步更新。
