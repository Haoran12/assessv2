# 分类一致性与权限边界说明（当前实现）

- 日期：2026-03-24
- 版本：v3.2

## 1. 分类一致性

## 1.1 分类来源

当前考核对象分类以场次对象分组为准：`assessment_object_groups`。

TopBar 的“考核对象类型”直接来源于当前场次对象分组。

## 1.2 一致性规则

1. 分组编码在单场次内唯一。
2. 对象通过 `groupCode` 归入分组。
3. 对象来源必须可在候选池（组织主数据）定位。
4. 对象属性变更在组织模块维护，场次只维护分组与排序。

## 2. 状态与可操作边界

1. `preparing` / `active`：允许场次配置写操作。
2. `completed`：场次配置与规则写操作全部只读。

## 3. 权限边界

1. Root：可操作任意组织场次（仍受 `completed` 只读限制）。
2. Admin：仅可操作授权组织场次（仍受 `completed` 只读限制）。

## 4. 规则操作说明

当前路由版本开放：规则查询、创建、更新、依赖检查。

- `GET /api/rules/files`
- `POST /api/rules/files`
- `PUT /api/rules/files/:id`
- `POST /api/rules/files/:id/dependency-check`

规则写操作同样遵循场次状态只读约束。

## 5. 验收建议

1. `completed` 场次下尝试改分组/对象，应失败。
2. 非授权 Admin 修改他组织场次，应失败。
3. `preparing` 与 `active` 状态下配置写入应成功。
