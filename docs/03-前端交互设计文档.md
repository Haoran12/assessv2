# AssessV2 前端交互设计文档（当前实现）

- 版本：v3.2
- 日期：2026-03-24
- 代码范围：`frontend/src`

## 1. 总体结构

技术栈：`Vue3 + TypeScript + Element Plus + Pinia + Vue Router`

布局：

- 左侧菜单
- 顶部上下文
- 中央业务视图

## 2. 生效页面

- `/overview` 考核主页
- `/assessment-management` 考核管理
- `/rules` 规则管理
- `/org` 组织架构
- `/system/users` 用户管理（Root）
- `/system/backup` 备份恢复
- `/system/audit` 审计日志
- `/system/settings` 系统设置（Root）

## 3. 顶部上下文

来源：`stores/context.ts`

字段：

- `sessionId`
- `periodCode`
- `objectGroupCode`

行为：

- 切换场次后自动刷新周期和对象分组。
- 周期/分组无效时自动回落到有效值。

## 4. 考核管理页（AssessmentView）

## 4.1 场次列表区

列表字段：

- ID
- 场次名称
- 年度
- 状态
- 目录名

状态列：

- 标签显示：准备中 / 进行中 / 已完成
- 下拉切换：`preparing` / `active` / `completed`

状态切换交互：

- 切换到 `completed` 先弹出确认框。
- 成功后刷新场次列表与详情。

## 4.2 场次内配置区

页签：

- 周期配置
- 对象分组配置
- 考核对象

写能力统一受 `canEditCurrentSession` 控制：

- 当 `status === completed` 时，按钮禁用且保存逻辑直接返回。

## 5. 考核主页（SystemOverviewView）

- 当前场次为 `completed` 时，评分编辑能力关闭。
- 页面给出只读提示，避免误操作。

## 6. 规则管理页（RulesView）

- 基于当前上下文加载规则与表达式上下文。
- `canEdit=false` 或场次只读时，仅允许查看。
- 保存后执行依赖检查并反馈错误/警告。

## 7. 前端 API 对齐

## 7.1 `assessment.ts`

包含：

- `updateAssessmentSessionStatus(sessionId, { status })`
- 周期/分组/对象维护接口
- 模块评分写入接口

## 7.2 `types/assessment.ts`

新增并使用：

- `AssessmentSessionStatus = "preparing" | "active" | "completed"`
- `status`
- `completedSnapshotPath?`
- `completedSnapshotCreatedAt?`

## 8. 前端只读策略

前端策略仅用于体验优化，后端为最终裁决：

1. 先禁用按钮与输入控件。
2. 保存函数入口再次判断 `canEditCurrentSession`。
3. 若后端返回业务错误，统一提示并保持页面数据一致性。
