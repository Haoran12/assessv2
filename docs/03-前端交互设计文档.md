## 4. 布局与全局交互

对应文件：`src/layouts/MainLayout.vue`

### 4.1 布局结构

- 左侧：一级菜单（10 项，按权限动态显示）
- 顶部：全局上下文 + 用户下拉菜单
- 主体：`<RouterView />`

### 4.2 全局上下文

顶部提供 3 个全局筛选器，供多页面共享：

- 年度（year）
- 周期（period）
- 考核分类（objectCategory）

状态由 `context` store 统一管理，并持久化到 `localStorage`。

### 4.3 会话与退出

- 支持退出登录、修改密码、退出系统（桌面端桥接）
- 支持未保存数据退出确认（`unsaved` store + beforeunload + Wails close guard）

---

## 5. 路由守卫与权限模型

对应文件：`src/router/index.ts`、`src/stores/app.ts`

### 5.1 鉴权逻辑

- 进入受保护路由前先尝试 `initializeSession()`
- 未登录访问受保护页面：跳转 `/login?redirect=...`
- 已登录但必须改密：强制跳转 `/change-password`

### 5.2 权限校验

- 路由 `meta.permission` 支持单权限或数组
- 使用 `appStore.hasAnyPermission()` 校验
- 无权限跳转 `/403`

### 5.3 权限匹配规则

- 支持精确权限：`rule:view`
- 支持全通配：`*`
- 支持前缀通配：例如 `rule:*`

## 6. 已实现页面交互说明

### 6.1 系统概览（`SystemOverviewView.vue`）

- 页面本体即“结果总览”
- 展示当前上下文下的考核结果列表（排名、对象、总分、等第、模块分）
- 支持关键字筛选
- 支持手动重算（有 `score:update` 权限时显示）
- 数据来源：
  - `assessment`：考核对象
  - `calc`：总分、排名、模块分、重算
  - `org`：组织/部门/人员辅助信息

### 6.2 周期管理（`AssessmentView.vue`）

- 年度列表、周期列表联动展示
- 支持创建年度（自动生成周期）
- 支持年度/周期状态流转
- 与全局上下文年度选择联动

### 6.3 组织架构（`OrganizationView.vue`）

- 左侧组织树 + 右侧多标签管理
- 组织管理、部门管理、分类管理、人员管理
- 支持人员调动与调动历史
- 支持按关键字/状态筛选

### 6.4 总分规则（`RulesView.vue`）

- 规则列表与模板列表
- 规则新建/编辑
- 模块配置、表达式配置、投票分组配置
- 前端校验：
  - 折算模块权重和必须为 1.0000
  - 分组权重和必须为 1.0000
  - 模块标识/名称必填

### 6.5 模块规则（`ModuleRulesView.vue`）

- 按“规则 + 模块”编辑细项
- 支持自定义表达式浮层编辑
- 支持投票分组维护
- 保存前做必填与表达式合法性检查

### 6.6 等第规则（`GradeRulesPlaceholderView.vue`）

- 当前为占位页
- 显示“等第规则页面暂未实现”

### 6.7 用户管理（`SystemUsersView.vue`）

- 用户列表：查询、分页、状态切换、重置密码
- 用户组管理：新增/编辑/删除
- 用户与用户组关系维护：分配组、主角色校验

### 6.8 备份恢复（`BackupManageView.vue`）

- 顶部操作：刷新、手动备份（需 `backup:update` 权限）
- 查询区：按备份类型筛选（手动/自动/导入前/恢复前）
- 提示区：展示恢复风险与 `CONFIRM_RESTORE` 二次确认要求
- 列表区：显示备份文件名、类型标签、大小、描述、时间
- 行操作：下载、恢复、删除（恢复/删除需权限）
- 支持分页与每页条数切换

### 6.9 审计日志（`AuditLogsView.vue`）

- 顶部操作：刷新
- 查询区：关键字、操作类型、目标类型、时间范围联合筛选
- 列表区：展示操作时间、操作人、动作类型、目标信息、IP、摘要
- 支持“详情”弹窗：查看字段差异（before/after）与原始详情 JSON
- 支持“回滚”操作（需 `audit:rollback` 权限），并在回滚后刷新列表
- 支持分页与每页条数切换

### 6.10 系统设置（`SystemSettingsView.vue`）

- 顶部操作：刷新、保存设置（需 `setting:update` 权限）
- 表单分区：基础设置、考核设置、安全设置、备份设置
- 字段类型覆盖输入框、数字输入、开关与多行 JSON 文本
- 保存前校验：
  - 时间字段必须为 `HH:mm`
  - JSON 配置必须可解析
- 保存成功后使用服务端返回值回填，确保页面与后端配置一致

### 6.11 认证页面

- 登录页（`LoginView.vue`）：登录、会话过期提示、重定向回跳
- 修改密码页（`ChangePasswordView.vue`）：强度与一致性校验
- 无权限页（`ForbiddenView.vue`）

## 7. 状态管理（Pinia）

### 7.1 `app` store

- 会话状态：token、当前用户、是否强制改密
- 登录、拉取 profile、修改密码、退出登录
- 权限判断：`hasPermission` / `hasAnyPermission`

### 7.2 `context` store

- 全局维度：年度、周期、考核分类
- 年度/周期/分类加载与初始化
- 本地持久化：`localStorage`

### 7.3 `unsaved` store

- 页面脏数据源登记与清理
- 用于窗口关闭保护和退出确认
