# AssessV2 前端交互设计文档（当前实现）

- 版本：v3.0
- 日期：2026-03-18
- 代码范围：`frontend/src`

## 1. 总体结构

前端基于 `Vue3 + TypeScript + Element Plus + Pinia + Vue Router`。

布局采用 `MainLayout`：

- 左侧：主菜单
- 顶部：全局上下文选择器 + 用户菜单
- 中央：业务页面容器（RouterView）

## 2. 路由与菜单

## 2.1 生效路由

- `/overview`：系统概览
- `/assessment-management`：考核管理
- `/org`：组织架构
- `/rules`：规则管理
- `/system/users`：用户管理（Root）
- `/system/backup`：备份恢复
- `/system/audit`：审计日志
- `/system/settings`：系统设置（Root）
- `/login`、`/change-password`、`/403`

## 2.2 路由守卫

- 未登录访问受保护页面 -> 跳转登录
- 必须改密用户 -> 强制进入改密页
- Root-only 页面进行角色校验
- 页面 `meta.permission` 进行权限点校验

## 3. 顶部全局上下文

上下文字段：

- `sessionId`（考核场次）
- `periodCode`（周期）
- `objectGroupCode`（考核对象类型）

状态来源：`stores/context.ts`

交互规则：

- 初次加载自动拉取场次列表。
- 若本地缓存场次不存在，自动回落第一个场次。
- 场次切换后自动加载该场次的周期与对象分组。
- 周期/分组若无效自动回落首项。

持久化键：

- `assessv2_context_session_id`
- `assessv2_context_period_code`
- `assessv2_context_group_code`

## 4. 页面交互设计

## 4.1 系统概览（`SystemOverviewView`）

功能：

- 展示当前用户、主角色、当前上下文状态。
- 提示系统采用“场次独立 + 规则绑定”模式。

## 4.2 考核管理（`AssessmentView`）

分为两块：

1. 场次列表与创建
2. 当前场次配置管理

### 4.2.1 场次列表区

- 列表字段：ID、场次名称、年份、组织、目录名
- 操作：刷新、进入管理
- 创建弹窗字段：组织、年份、场次名称、说明
- 创建默认名：`当前年份 + 组织名 + 考核`（用户可改）

### 4.2.2 周期配置区

- 可编辑周期编码和名称
- 支持新增、删除、整表保存
- 保存接口：`PUT /api/assessment/sessions/:id/periods`

### 4.2.3 对象分组配置区

- 字段：对象类型（团体/个人）、分组编码、分组名称
- 支持新增、删除、整表保存
- 保存接口：`PUT /api/assessment/sessions/:id/object-groups`

### 4.2.4 考核对象区

- 列出当前场次对象集合
- 可删除对象项
- 支持“新增对象”弹窗：
  - 从候选池选择实体
  - 自动推荐对象类型与分组
- 支持保存对象清单
- 支持“重置为默认对象”

相关接口：

- `GET /objects`
- `GET /object-candidates`
- `PUT /objects`
- `POST /objects/reset-default`

## 4.3 规则管理（`RulesView`）

页面左右分栏：

- 左栏：规则文件库
- 右栏：结构化规则编辑

### 4.3.1 左栏规则文件库

- 显示规则名称、类型标签（基础/拷贝）
- 支持按开关显示隐藏规则
- 操作按钮：
  - 绑定
  - 隐藏/恢复（Admin 对基础规则）
  - 删除（按权限）

### 4.3.2 绑定交互

绑定前依赖 TopBar 完整上下文：

- 场次已选
- 周期已选
- 对象分组已选

绑定动作：

- 调用 `POST /api/rules/bindings/select`
- 成功后提示“已绑定并自动创建组织拷贝”
- 自动刷新文件库与绑定列表
- 自动选中新生成的拷贝规则

### 4.3.3 结构化规则编辑

编辑内容：

- 规则元信息（名称、说明）
- 分数计算（两级）
- 等第划分

分数计算区特性：

- 常规模块可新增、详情编辑、删除
- 额外加减分模块不可删除
- 显示总权重与总分公式

模块详情弹窗支持：

- 模块名称、权重、计分方式
- 计分方式配置（直接录入/投票/脚本）
- 子模块增删改

编辑校验：

- 至少一个常规模块
- 必须且仅有一个额外加减分模块
- 常规模块权重大于 0
- 子模块名称不能为空
- 等第至少一条且范围合法

保存接口：

- `PUT /api/rules/files/:id`
- 仅在 `canEdit=true` 时允许

## 4.4 组织架构（`OrganizationView`）

- 组织、部门、员工管理
- 支持树形浏览与列表编辑
- 为场次默认对象和候选对象提供源数据

## 4.5 系统管理页面

- 用户管理：用户、角色组、分配关系
- 备份恢复：备份列表、手动备份、下载、恢复
- 审计日志：过滤、详情、回滚
- 系统设置：配置项查看/更新

## 5. API 调用层

目录：`frontend/src/api`

核心文件：

- `assessment.ts`
- `rules.ts`
- `org.ts`
- `system-admin.ts`
- `http.ts`

`http.ts` 统一处理：

- Bearer Token 注入
- 响应解包与业务错误提示
- 会话失效处理

## 6. 状态管理

## 6.1 `app` store

- 用户会话
- 角色与权限判断
- 登录/登出/改密

## 6.2 `context` store

- 全局业务上下文
- 场次详情派生（当前周期、当前对象分组）
- 场次相关数据刷新方法

## 7. 可用性与反馈

统一使用 `ElMessage` 提示：

- 成功：保存、绑定、删除、重置
- 警告：上下文缺失、校验失败
- 错误：接口失败或权限不足

## 8. 当前限制

- 规则绑定依赖 TopBar 三项上下文，缺失则无法绑定。
- 规则编辑仅支持拷贝规则（基础规则不可直接编辑）。
- 页面中仍保留少量历史 API 文件（如 `calc.ts`），不在当前主流程路由使用。
