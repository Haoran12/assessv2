# AssessV2 多级考核对象划分与规则复用设计（现行实现）

- 版本：v3.0
- 日期：2026-03-18

## 1. 设计背景

系统已从“全局分类 + 年度规则”切换为“场次分组 + 绑定拷贝规则”。

核心诉求：

- 支持组织层级下对象自动生成与人工调整。
- 支持同一场次在不同周期/对象分组下绑定不同规则。
- 支持组织间共享规则模板但互不污染。

## 2. 对象划分模型

## 2.1 对象实体来源

所有考核对象均来自组织主数据：

- 组织（organization）
- 部门（department）
- 员工（employee）

场次对象不维护独立主数据，只保存“引用快照 + 分组归属”。

## 2.2 默认对象生成策略

创建场次时自动生成：

1. 本组织全部部门（团体）
2. 本组织全部人员（个人）
3. 下一级组织整体（团体）
4. 下一级组织领导班子成员（个人）

对象层级关系：

- 本组织员工默认挂到所属部门对象下
- 下级组织领导成员默认挂到所属下级组织对象下

## 2.3 场次对象分组策略

分组是场次内可配置维度，不依赖全局模板。

默认分组（可改名、可增删）：

- 团体：`dept`、`child_org`
- 个人：`dept_main`、`dept_deputy`、`general_staff`、`child_leadership_main`、`child_leadership_deputy`

## 3. 规则复用模型

## 3.1 绑定维度

规则绑定的最小维度为：

- `assessmentId`
- `periodCode`
- `objectGroupCode`
- `organizationId`

即：

`场次 + 周期 + 对象分组 + 组织 -> 规则文件`

## 3.2 绑定即拷贝

当用户从规则文件库选中一条规则进行绑定时：

1. 复制源规则内容为新规则文件
2. 标记 `isCopy=true`
3. 设置 `sourceRuleId` 与 `ownerOrgId`
4. 绑定关系指向新拷贝

收益：

- 同源规则可被多个组织复用
- 每个组织后续修改互不影响

## 3.3 绑定更新策略

若同维度绑定已存在：

- 不修改历史源规则
- 直接将绑定指向新创建拷贝

说明：

- 该策略保证每次“重新选择规则”都形成新拷贝快照。

## 4. 规则内容结构

结构化规则内容包含两部分：

- 分数计算（`scoreModules`）
- 等第划分（`gradeRules`）

分数计算模块要求：

- 常规模块支持两级结构
- 额外加减分模块固定存在，不可删除

## 5. 权限与复用边界

## 5.1 查看范围

- Root 与 Admin 均可查看当前场次规则文件。

## 5.2 操作范围

基础规则：

- Root 可新建与删除
- Admin 不可删除，可隐藏/恢复显示

拷贝规则：

- Root 可编辑/删除全部
- Admin 仅可编辑/删除自己组织拷贝

## 6. 与组织架构一致性原则

- 场次对象仅做“选择与分组”，不做实体属性修改。
- 对象名称、组织归属、部门归属等主属性变更应在组织模块完成。
- 场次对象候选池实时从组织数据生成，保证来源一致。

## 7. 现行实现下的典型流程

1. Root 或 Admin 创建场次。
2. 系统生成默认周期、默认对象分组、默认对象。
3. 在“规则管理”中选定上下文（场次/周期/对象分组）。
4. 从规则文件库绑定规则，系统自动生成组织拷贝。
5. 编辑拷贝规则并保存，仅影响本组织本拷贝。

## 8. 已知约束

- 对象分组编码在场次内必须唯一。
- 同一场次对象目标（`targetType + targetId`）不可重复。
- 绑定前必须存在目标周期与对象分组。

## 9. 后续可演进方向

- 增加规则版本链与历史回溯。
- 增加对象快照差异分析（组织变更影响）。
- 提供“按模板批量绑定”能力。

---

本设计是当前已实现的对象划分与规则复用机制说明，不再沿用旧 `object_category` 规则驱动模式。
