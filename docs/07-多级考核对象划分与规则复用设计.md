# AssessV2 对象与规则复用说明（当前实现）

- 版本：v3.2
- 日期：2026-03-24

## 1. 对象来源模型

场次对象全部来自组织主数据：

- organization
- department
- employee

场次内保存的是对象快照与分组归属，不维护独立组织主数据。

## 2. 场次对象分组

场次分组由 `assessment_object_groups` 管理，场次内独立维护。

默认分组：

- 团体：`dept`、`child_org`
- 个人：`dept_main`、`dept_deputy`、`general_staff`、`child_leadership_main`、`child_leadership_deputy`

## 3. 场次状态与对象重置

## 3.1 `preparing`

- 重置对象时基于当前组织架构重新生成。

## 3.2 `active`

- 重置对象时优先使用场次对象快照。

## 3.3 `completed`

- 场次只读，不允许对象重置或维护。

## 4. 规则复用与生效

规则文件由 `rule_files` 管理，规则内容按场次上下文（周期/对象分组）组织。

- 编辑入口在规则管理页。
- 保存后进行依赖检查（`/rules/files/:id/dependency-check`）。
- `completed` 场次下规则写入被拒绝。

## 5. 一致性约束

1. 对象分组编码在场次内唯一。
2. 同场次 `targetType + targetId` 唯一。
3. 周期编码统一大写。
4. 规则编辑依赖有效场次上下文。

## 6. 典型流程

1. 创建场次（默认 `preparing`）
2. 维护周期/对象分组/对象
3. 切到 `active` 后开展评分与规则迭代
4. 切到 `completed` 封板并归档快照
