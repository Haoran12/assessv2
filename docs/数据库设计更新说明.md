# 数据库设计更新说明（场次化重构）

- 日期：2026-03-18
- 文档版本：v3.0
- 适用迁移：`business 0002`、`accounts 1002`

## 1. 本次更新目标

数据库从“年度驱动”重构为“场次驱动”，并同步规则文件拷贝绑定机制。

## 2. 业务库变更摘要

## 2.1 新增核心表

- `assessment_sessions`
- `assessment_session_periods`
- `assessment_object_groups`
- `assessment_session_objects`
- `rule_files`
- `rule_file_hides`
- `assessment_rule_bindings`

## 2.2 移除旧表

- `assessment_years`
- `assessment_periods`
- `assessment_objects`
- `assessment_categories`
- `assessment_rules`
- `rule_bindings`
- `rule_templates`
- `score_modules`
- `vote_groups`
- `direct_scores`
- `vote_tasks`
- `vote_records`
- `extra_points`
- `calculated_scores`
- `calculated_module_scores`
- `rankings`
- `assessment_object_user_links`

## 2.3 关键约束变化

- `assessment_sessions` 仅对 `assessment_name` 做唯一约束。
- 未引入 `(year, organization_id)` 唯一限制。
- 绑定唯一键改为：
  - `assessment_id + period_code + object_group_code + organization_id`

## 3. 文件系统路径变化

创建场次时自动创建：

- `data/{assessment_name}/`
- `data/rules/{assessment_name}/`

规则元数据与文件路径双写：

- 元数据在 `rule_files`
- 内容文件在 `file_path`

## 4. 账号库变更摘要

- 迁移 `1002_remove_user_real_name` 删除 `users.real_name`。
- 账号模型继续使用：
  - `users`
  - `roles`
  - `user_roles`
  - `user_organizations`
  - `user_permission_bindings`

## 5. 与业务行为联动的更新点

- 场次创建后默认写入周期、对象分组、对象清单。
- 规则绑定改为“每次创建拷贝并更新绑定”。
- Admin 的规则隐藏能力通过 `rule_file_hides` 实现。

## 6. 升级注意事项

- 本次为重建式迁移，不保证旧年度业务数据兼容。
- 应先备份旧库，再执行迁移。
- 若存在历史依赖旧表的脚本，需同步重写。

## 7. 快速核查清单

```sql
-- 1) 检查新场次主表
SELECT name FROM sqlite_master WHERE type='table' AND name='assessment_sessions';

-- 2) 检查旧年度表是否已移除
SELECT name FROM sqlite_master WHERE type='table' AND name='assessment_years';

-- 3) 检查规则绑定表
SELECT name FROM sqlite_master WHERE type='table' AND name='assessment_rule_bindings';
```

预期：1、3 存在；2 不存在。

---

如后续新增表或调整约束，需同时更新《02-数据库设计文档》与本更新说明。
