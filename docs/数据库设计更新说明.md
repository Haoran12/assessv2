# 数据库设计更新说明（截至 2026-03-24）

- 文档版本：v3.2
- 适用迁移：`business 0001~0010`、`accounts 1001~1002`

## 1. 更新概要

当前数据库已完全采用场次中心模型，并支持场次三状态治理。

## 2. 关键更新项

## 2.1 场次状态字段（business 0010）

在 `assessment_sessions` 新增：

- `status`（默认 `preparing`）
- `completed_snapshot_path`
- `completed_snapshot_created_at`

并新增索引：

- `idx_assessment_sessions_status`

## 2.2 场次周期绑定键（business 0004）

- 周期记录支持规则绑定键，用于共享规则周期逻辑。

## 2.3 评分表与明细扩展（business 0006 / 0009）

- 新增对象模块评分表与评分明细字段。

## 2.4 备份与审计结构升级（business 0007 / 0008）

- 备份记录支持组织逻辑包扩展字段。
- 审计日志结构升级并统一。

## 3. 当前一致性要求

1. 场次状态值仅允许：`preparing`、`active`、`completed`。
2. `completed` 场次需具备完成快照元数据。
3. 所有写操作由服务层依据状态与权限共同判定。

## 4. 快速核查

```sql
-- 1) 场次状态字段
PRAGMA table_info('assessment_sessions');

-- 2) 状态索引
SELECT name FROM sqlite_master
WHERE type='index' AND name='idx_assessment_sessions_status';

-- 3) 迁移版本检查（示意）
SELECT * FROM schema_migrations ORDER BY version;
```
