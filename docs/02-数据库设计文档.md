## 1. 数据库概述

### 1.1 基本信息

- **数据库类型**: SQLite 3.45+
- **字符编码**: UTF-8
- **外键约束**: 启用（PRAGMA foreign_keys = ON）
- **日志模式**: WAL（Write-Ahead Logging）
- **同步模式**: NORMAL
- **数据精度**:
  - 分数存储：DECIMAL(10, 6) - 保留 6 位小数
  - 权重存储：DECIMAL(5, 4) - 保留 4 位小数
  - 时间戳：INTEGER（Unix timestamp，秒级）

### 1.2 设计原则

1. **数据完整性**：使用外键约束保证引用完整性
2. **数据隔离**：年度数据完全隔离，通过 year_id 关联
3. **审计追踪**：关键表记录 created_by, created_at, updated_by, updated_at
4. **软删除**：重要数据使用 deleted_at 字段标记删除（NULL 表示未删除）
5. **性能优化**：合理使用索引，避免过度索引
6. **扩展性**：支持用户自定义职级分类

### 1.3 表分类

本系统共设计 **25 张表**，按功能模块分类如下：

- **用户与权限模块**（4 张）：users, roles, user_roles, user_organizations
- **组织架构模块**（5 张）：organizations, departments, position_levels, employees, employee_history
- **考核管理模块**（3 张）：assessment_years, assessment_periods, assessment_objects
- **规则配置模块**（3 张）：assessment_rules, score_modules, vote_groups
- **分数数据模块**（4 张）：direct_scores, vote_tasks, vote_records, extra_points
- **计算结果模块**（2 张）：calculated_scores, rankings
- **备份审计模块**（2 张）：backups, audit_logs
- **系统配置模块**（2 张）：system_settings, rule_templates

## 2. 表结构设计

### 2.1 用户与权限模块

#### 2.1.1 users - 用户表

存储系统用户信息和认证数据。

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,  -- bcrypt hash, cost=10
    real_name VARCHAR(100) NOT NULL,
    email VARCHAR(100),
    phone VARCHAR(20),
    status VARCHAR(20) NOT NULL DEFAULT 'active',  -- active, inactive, locked
    must_change_password BOOLEAN NOT NULL DEFAULT 0,  -- 首次登录强制修改密码
    last_login_at INTEGER,    -- Unix timestamp
    last_login_ip VARCHAR(50),
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL,
    deleted_at INTEGER,       -- 软删除
    CONSTRAINT chk_user_status CHECK (status IN ('active', 'inactive', 'locked'))
);
```

**字段说明**：
- `password_hash`: 使用 bcrypt 加密，cost=10
- `status`: active（正常）、inactive（停用）、locked（锁定）
- `must_change_password`: 首次登录或密码重置后需强制修改
- **注意**: 用户的角色和组织关系通过 user_roles 和 user_organizations 表关联，支持多角色多组织

#### 2.1.2 roles - 角色表

定义系统角色和权限。

```sql
CREATE TABLE roles (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    role_code VARCHAR(50) NOT NULL UNIQUE,
    role_name VARCHAR(100) NOT NULL,
    description TEXT,
    permissions TEXT NOT NULL,  -- JSON 格式存储权限列表
    is_system BOOLEAN NOT NULL DEFAULT 0,  -- 系统预置角色不可删除
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);
```

**字段说明**：
- `role_code`: super_admin, group_admin, company_admin, dept_admin, viewer
- `permissions`: JSON 格式，如 `["user:create", "user:update", "score:input"]`
- `is_system`: 系统预置角色（超级管理员等）不可删除

#### 2.1.3 user_roles - 用户角色关联表

支持用户拥有多个角色，实现灵活的权限管理。

```sql
CREATE TABLE user_roles (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    role_id INTEGER NOT NULL,
    is_primary BOOLEAN NOT NULL DEFAULT 0,  -- 是否为主要角色
    created_by INTEGER,
    created_at INTEGER NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (role_id) REFERENCES roles(id) ON DELETE CASCADE,
    FOREIGN KEY (created_by) REFERENCES users(id),
    UNIQUE(user_id, role_id)
);
```

**字段说明**：
- `is_primary`: 标记用户的主要角色，用于默认权限判断
- 一个用户可以有多个角色，权限取并集
- 例如：用户可以同时是"集团管理员"和"企业管理员"

**设计特性**（v2.1 新增）：
- ✅ **多角色支持**：用户可以拥有多个系统角色，不再局限于单一角色
- ✅ **权限并集**：用户的最终权限 = 所有角色权限的并集
- ✅ **主要角色标记**：通过 is_primary 标记主要角色，用于默认权限判断和UI显示
- 📝 详细使用场景和权限判断逻辑请参考：[数据库设计更新说明.md](./数据库设计更新说明.md#权限判断逻辑)

#### 2.1.4 user_organizations - 用户组织关联表

支持用户负责多个组织（集团/企业/部门）。

```sql
CREATE TABLE user_organizations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    organization_type VARCHAR(20) NOT NULL,  -- group, company, department
    organization_id INTEGER NOT NULL,  -- 对应的组织/企业/部门 ID
    role_in_org VARCHAR(50),  -- 在该组织中的角色：leader, admin, member
    is_primary BOOLEAN NOT NULL DEFAULT 0,  -- 是否为主要负责组织
    created_by INTEGER,
    created_at INTEGER NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (created_by) REFERENCES users(id),
    CONSTRAINT chk_org_type CHECK (organization_type IN ('group', 'company', 'department')),
    UNIQUE(user_id, organization_type, organization_id)
);
```

**字段说明**：
- `organization_type`: group（集团）、company（企业）、department（部门）
- `organization_id`: 根据 organization_type 指向不同的表
  - group: 指向 organizations 表（org_type='group'）
  - company: 指向 organizations 表（org_type='company'）
  - department: 指向 departments 表
- `role_in_org`: 在该组织中的角色
  - leader: 负责人（如企业总经理、部门经理）- 拥有该组织的完全管理权限
  - admin: 管理员（有管理权限）- 可以管理数据但不能修改组织结构
  - member: 成员（普通成员）- 只能查看数据
- `is_primary`: 标记用户的主要负责组织

**设计特性**（v2.1 新增）：
- ✅ **多组织支持**：用户可以同时负责多个集团/企业/部门
- ✅ **灵活的组织角色**：用户可以在不同组织中担任不同角色
- ✅ **数据范围控制**：用户可以访问所有关联组织的数据
- ✅ **主要组织标记**：通过 is_primary 标记主要负责组织

**典型使用场景**：
- 场景1：用户张三同时负责企业A和企业B（两个企业的总经理）
- 场景2：用户李四在集团是成员，在部门A是管理员，在部门B是负责人
- 场景3：用户王五同时管理多个部门的考核工作
- 📝 更多场景示例和查询方法请参考：[数据库设计更新说明.md](./数据库设计更新说明.md#使用场景示例)

### 2.2 组织架构模块

#### 2.2.1 organizations - 组织表

存储集团和权属企业信息。

```sql
CREATE TABLE organizations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    org_code VARCHAR(50) NOT NULL UNIQUE,
    org_name VARCHAR(200) NOT NULL,
    org_type VARCHAR(20) NOT NULL,  -- group, company
    parent_id INTEGER,  -- 上级组织（集团的 parent_id 为 NULL）
    leader_id INTEGER,  -- 负责人（员工 ID）
    sort_order INTEGER NOT NULL DEFAULT 0,
    status VARCHAR(20) NOT NULL DEFAULT 'active',  -- active, inactive
    created_by INTEGER,
    created_at INTEGER NOT NULL,
    updated_by INTEGER,
    updated_at INTEGER NOT NULL,
    deleted_at INTEGER,
    FOREIGN KEY (parent_id) REFERENCES organizations(id),
    FOREIGN KEY (leader_id) REFERENCES employees(id),
    FOREIGN KEY (created_by) REFERENCES users(id),
    FOREIGN KEY (updated_by) REFERENCES users(id)
);
```

**字段说明**：
- `org_type`: group（集团，唯一）、company（权属企业）
- `parent_id`: 集团的 parent_id 为 NULL，企业的 parent_id 指向集团或上级企业
- `status`: active（启用）、inactive（停用，不参与考核但保留历史数据）

#### 2.2.2 departments - 部门表

存储集团部门和企业部门信息。

```sql
CREATE TABLE departments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    dept_code VARCHAR(50) NOT NULL,
    dept_name VARCHAR(200) NOT NULL,
    organization_id INTEGER NOT NULL,  -- 所属组织（集团或企业）
    parent_dept_id INTEGER,  -- 上级部门（支持部门层级）
    leader_id INTEGER,  -- 部门负责人（员工 ID）
    sort_order INTEGER NOT NULL DEFAULT 0,
    status VARCHAR(20) NOT NULL DEFAULT 'active',  -- active, inactive
    created_by INTEGER,
    created_at INTEGER NOT NULL,
    updated_by INTEGER,
    updated_at INTEGER NOT NULL,
    deleted_at INTEGER,
    FOREIGN KEY (organization_id) REFERENCES organizations(id),
    FOREIGN KEY (parent_dept_id) REFERENCES departments(id),
    FOREIGN KEY (leader_id) REFERENCES employees(id),
    FOREIGN KEY (created_by) REFERENCES users(id),
    FOREIGN KEY (updated_by) REFERENCES users(id),
    UNIQUE(organization_id, dept_code)
);
```

**字段说明**：
- `organization_id`: 所属集团或企业
- `parent_dept_id`: 支持部门层级结构（可选）
- 同一组织内部门代码唯一

#### 2.2.3 position_levels - 职级定义表

存储人员职级分类定义，支持用户自定义。

```sql
CREATE TABLE position_levels (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    level_code VARCHAR(50) NOT NULL UNIQUE,
    level_name VARCHAR(100) NOT NULL,
    description TEXT,
    is_system BOOLEAN NOT NULL DEFAULT 0,  -- 系统预置职级不可删除
    is_for_assessment BOOLEAN NOT NULL DEFAULT 1,  -- 是否用于考核对象分类
    sort_order INTEGER NOT NULL DEFAULT 0,
    status VARCHAR(20) NOT NULL DEFAULT 'active',  -- active, inactive
    created_by INTEGER,
    created_at INTEGER NOT NULL,
    updated_by INTEGER,
    updated_at INTEGER NOT NULL,
    FOREIGN KEY (created_by) REFERENCES users(id),
    FOREIGN KEY (updated_by) REFERENCES users(id)
);
```

**字段说明**：
- `level_code`: 职级代码，如 group_leader, company_leader, manager_main, manager_deputy, staff
- `is_system`: 系统预置职级（集团高层、企业高层等）不可删除
- `is_for_assessment`: 是否用于考核对象分类（用户可自定义职级但不一定用于考核）
- `status`: active（启用）、inactive（停用）

**系统预置职级**：
- group_leader: 集团高层领导
- company_leader: 权属企业高层领导
- manager_main: 正职管理人员
- manager_deputy: 副职管理人员
- staff: 一般人员

#### 2.2.4 employees - 员工表

存储人员信息。

```sql
CREATE TABLE employees (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    emp_code VARCHAR(50) NOT NULL UNIQUE,  -- 工号
    emp_name VARCHAR(100) NOT NULL,
    organization_id INTEGER NOT NULL,  -- 所属组织
    department_id INTEGER,  -- 所属部门（高层领导可能不属于部门）
    position_level_id INTEGER NOT NULL,  -- 职级
    position_title VARCHAR(100),  -- 职务名称（如：总经理、部门经理）
    hire_date DATE,  -- 入职日期
    status VARCHAR(20) NOT NULL DEFAULT 'active',  -- active, inactive
    created_by INTEGER,
    created_at INTEGER NOT NULL,
    updated_by INTEGER,
    updated_at INTEGER NOT NULL,
    deleted_at INTEGER,
    FOREIGN KEY (organization_id) REFERENCES organizations(id),
    FOREIGN KEY (department_id) REFERENCES departments(id),
    FOREIGN KEY (position_level_id) REFERENCES position_levels(id),
    FOREIGN KEY (created_by) REFERENCES users(id),
    FOREIGN KEY (updated_by) REFERENCES users(id)
);
```

**字段说明**：
- `emp_code`: 工号，全局唯一
- `department_id`: 高层领导可能不属于具体部门，可为 NULL
- `position_level_id`: 关联职级定义表
- `status`: active（在职）、inactive（离职）

#### 2.2.5 employee_history - 员工变动历史表

记录员工的组织、部门、职级变动历史。

```sql
CREATE TABLE employee_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    employee_id INTEGER NOT NULL,
    change_type VARCHAR(50) NOT NULL,  -- transfer, promotion, demotion, position_change
    old_organization_id INTEGER,
    new_organization_id INTEGER,
    old_department_id INTEGER,
    new_department_id INTEGER,
    old_position_level_id INTEGER,
    new_position_level_id INTEGER,
    old_position_title VARCHAR(100),
    new_position_title VARCHAR(100),
    change_reason TEXT,
    effective_date DATE NOT NULL,  -- 生效日期
    created_by INTEGER,
    created_at INTEGER NOT NULL,
    FOREIGN KEY (employee_id) REFERENCES employees(id),
    FOREIGN KEY (old_organization_id) REFERENCES organizations(id),
    FOREIGN KEY (new_organization_id) REFERENCES organizations(id),
    FOREIGN KEY (old_department_id) REFERENCES departments(id),
    FOREIGN KEY (new_department_id) REFERENCES departments(id),
    FOREIGN KEY (old_position_level_id) REFERENCES position_levels(id),
    FOREIGN KEY (new_position_level_id) REFERENCES position_levels(id),
    FOREIGN KEY (created_by) REFERENCES users(id)
);
```

**字段说明**：
- `change_type`: transfer（调动）、promotion（晋升）、demotion（降职）、position_change（职务变更）
- 记录变更前后的组织、部门、职级信息
- `effective_date`: 变更生效日期

### 2.3 考核管理模块

#### 2.3.1 assessment_years - 考核年度表

存储考核年度信息。

```sql
CREATE TABLE assessment_years (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    year INTEGER NOT NULL UNIQUE,  -- 年度，如 2024
    year_name VARCHAR(100) NOT NULL,  -- 年度名称，如 "2024年度考核"
    status VARCHAR(20) NOT NULL DEFAULT 'preparing',  -- preparing, active, ended
    start_date DATE,
    end_date DATE,
    description TEXT,
    created_by INTEGER,
    created_at INTEGER NOT NULL,
    updated_by INTEGER,
    updated_at INTEGER NOT NULL,
    FOREIGN KEY (created_by) REFERENCES users(id),
    FOREIGN KEY (updated_by) REFERENCES users(id)
);
```

**字段说明**：
- `year`: 年度数字，唯一
- `status`: preparing（筹备中）、active（进行中）、ended（已结束）
- 年度间数据完全隔离

#### 2.3.2 assessment_periods - 考核周期表

存储考核周期信息（季度和年终）。

```sql
CREATE TABLE assessment_periods (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    year_id INTEGER NOT NULL,
    period_code VARCHAR(20) NOT NULL,  -- Q1, Q2, Q3, Q4, YEAR_END
    period_name VARCHAR(100) NOT NULL,  -- 第一季度、第二季度、年终考核
    status VARCHAR(20) NOT NULL DEFAULT 'not_started',  -- not_started, active, ended, locked
    start_date DATE,
    end_date DATE,
    created_by INTEGER,
    created_at INTEGER NOT NULL,
    updated_by INTEGER,
    updated_at INTEGER NOT NULL,
    FOREIGN KEY (year_id) REFERENCES assessment_years(id) ON DELETE CASCADE,
    FOREIGN KEY (created_by) REFERENCES users(id),
    FOREIGN KEY (updated_by) REFERENCES users(id),
    UNIQUE(year_id, period_code)
);
```

**字段说明**：
- `period_code`: Q1, Q2, Q3, Q4, YEAR_END
- `status`: not_started（未开始）、active（进行中）、ended（已结束）、locked（已锁定，不可修改）
- 每个年度自动创建 5 个周期

#### 2.3.3 assessment_objects - 考核对象表

存储参与考核的对象（团体或个人）。

```sql
CREATE TABLE assessment_objects (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    year_id INTEGER NOT NULL,
    object_type VARCHAR(20) NOT NULL,  -- team, individual
    object_category VARCHAR(50) NOT NULL,  -- 对象分类
    target_id INTEGER NOT NULL,  -- 目标 ID（组织/部门/员工）
    target_type VARCHAR(20) NOT NULL,  -- organization, department, employee
    object_name VARCHAR(200) NOT NULL,  -- 冗余存储，便于查询
    parent_object_id INTEGER,  -- 所属团体（个人考核对象关联团体）
    is_active BOOLEAN NOT NULL DEFAULT 1,
    created_by INTEGER,
    created_at INTEGER NOT NULL,
    updated_by INTEGER,
    updated_at INTEGER NOT NULL,
    FOREIGN KEY (year_id) REFERENCES assessment_years(id) ON DELETE CASCADE,
    FOREIGN KEY (parent_object_id) REFERENCES assessment_objects(id),
    FOREIGN KEY (created_by) REFERENCES users(id),
    FOREIGN KEY (updated_by) REFERENCES users(id),
    UNIQUE(year_id, target_type, target_id)
);
```

**字段说明**：
- `object_type`: team（团体）、individual（个人）
- `object_category`:
  - 团体：group_dept（集团部门）、company（权属企业）、company_dept（企业部门）
  - 个人：group_leader（集团高层）、company_leader（企业高层）、manager_main（正职管理人员）、manager_deputy（副职管理人员）、staff（一般人员）
- `target_id` + `target_type`: 指向实际的组织/部门/员工
- `parent_object_id`: 个人考核对象关联所属团体考核对象

### 2.4 规则配置模块

#### 2.4.1 assessment_rules - 考核规则表

存储考核规则配置。

```sql
CREATE TABLE assessment_rules (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    year_id INTEGER NOT NULL,
    period_code VARCHAR(20) NOT NULL,  -- Q1, Q2, Q3, Q4, YEAR_END
    object_type VARCHAR(20) NOT NULL,  -- team, individual
    object_category VARCHAR(50) NOT NULL,  -- 对象分类
    rule_name VARCHAR(200) NOT NULL,
    description TEXT,
    is_active BOOLEAN NOT NULL DEFAULT 1,
    created_by INTEGER,
    created_at INTEGER NOT NULL,
    updated_by INTEGER,
    updated_at INTEGER NOT NULL,
    FOREIGN KEY (year_id) REFERENCES assessment_years(id) ON DELETE CASCADE,
    FOREIGN KEY (created_by) REFERENCES users(id),
    FOREIGN KEY (updated_by) REFERENCES users(id),
    UNIQUE(year_id, period_code, object_type, object_category)
);
```

**字段说明**：
- 规则按年度、周期、对象类型、对象分类配置
- 同一配置维度只能有一条规则

#### 2.4.2 score_modules - 分数模块表

存储考核规则下的分数模块配置。

```sql
CREATE TABLE score_modules (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    rule_id INTEGER NOT NULL,
    module_code VARCHAR(50) NOT NULL,  -- direct, vote, extra
    module_name VARCHAR(100) NOT NULL,
    weight DECIMAL(5, 4) NOT NULL,  -- 权重，如 0.7000
    max_score DECIMAL(10, 6),  -- 最高分（直接录入模块使用）
    calculation_method VARCHAR(50),  -- 计算方法：average, weighted_average, sum
    sort_order INTEGER NOT NULL DEFAULT 0,
    is_active BOOLEAN NOT NULL DEFAULT 1,
    created_by INTEGER,
    created_at INTEGER NOT NULL,
    updated_by INTEGER,
    updated_at INTEGER NOT NULL,
    FOREIGN KEY (rule_id) REFERENCES assessment_rules(id) ON DELETE CASCADE,
    FOREIGN KEY (created_by) REFERENCES users(id),
    FOREIGN KEY (updated_by) REFERENCES users(id),
    UNIQUE(rule_id, module_code)
);
```

**字段说明**：
- `module_code`: direct（直接录入）、vote（投票）、extra（权重外加减分）
- `weight`: 模块权重，同一规则下所有模块权重之和应为 1.0000
- `max_score`: 直接录入模块的最高分限制
- `calculation_method`: 投票模块的计算方法（平均值、加权平均、求和等）

#### 2.4.3 vote_groups - 投票分组表

存储投票模块下的投票分组配置。

```sql
CREATE TABLE vote_groups (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    module_id INTEGER NOT NULL,  -- 所属分数模块
    group_code VARCHAR(50) NOT NULL,
    group_name VARCHAR(100) NOT NULL,
    weight DECIMAL(5, 4) NOT NULL,  -- 分组权重
    voter_type VARCHAR(50) NOT NULL,  -- 投票人类型
    voter_scope TEXT,  -- 投票人范围（JSON 格式）
    max_score DECIMAL(10, 6) NOT NULL,  -- 最高分
    sort_order INTEGER NOT NULL DEFAULT 0,
    is_active BOOLEAN NOT NULL DEFAULT 1,
    created_by INTEGER,
    created_at INTEGER NOT NULL,
    updated_by INTEGER,
    updated_at INTEGER NOT NULL,
    FOREIGN KEY (module_id) REFERENCES score_modules(id) ON DELETE CASCADE,
    FOREIGN KEY (created_by) REFERENCES users(id),
    FOREIGN KEY (updated_by) REFERENCES users(id),
    UNIQUE(module_id, group_code)
);
```

**字段说明**：
- `voter_type`: group_leader（集团领导）、company_leader（企业领导）、dept_leader（部门领导）、peer（同级）、subordinate（下级）
- `voter_scope`: JSON 格式，定义投票人范围，如 `{"organization_ids": [1,2,3]}` 或 `{"department_ids": [5,6]}`
- `weight`: 分组权重，同一模块下所有分组权重之和应为 1.0000

## 4. 外键关系

### 4.1 关系图说明

本系统采用严格的外键约束，确保数据引用完整性。主要关系如下：

### 4.2 用户与权限关系

```
roles (1) ----< (N) user_roles >----< (N) users
users (1) ----< (N) user_organizations
organizations (1) ----< (N) user_organizations
departments (1) ----< (N) user_organizations
```

**说明**：
- 用户通过 user_roles 表关联多个角色（多对多关系）
- 用户通过 user_organizations 表关联多个组织（多对多关系）
- 用户可以在不同组织中担任不同角色
- 支持标记主要角色和主要负责组织

### 4.3 组织架构关系

```
organizations (1) ----< (N) organizations (自关联，parent_id)
organizations (1) ----< (N) departments
departments (1) ----< (N) departments (自关联，parent_dept_id)
position_levels (1) ----< (N) employees
organizations (1) ----< (N) employees
departments (1) ----< (N) employees
employees (1) ----< (N) employee_history
```

**说明**：
- 组织支持层级结构（集团 > 企业）
- 部门支持层级结构（一级部门 > 二级部门）
- 员工关联组织、部门、职级
- 员工变动历史记录所有变更

### 4.4 考核管理关系

```
assessment_years (1) ----< (N) assessment_periods
assessment_years (1) ----< (N) assessment_objects
assessment_objects (1) ----< (N) assessment_objects (自关联，parent_object_id)
```

**说明**：
- 每个年度包含多个考核周期（Q1-Q4, YEAR_END）
- 每个年度包含多个考核对象
- 个人考核对象可关联团体考核对象（parent_object_id）

### 4.5 规则配置关系

```
assessment_years (1) ----< (N) assessment_rules
assessment_rules (1) ----< (N) score_modules
score_modules (1) ----< (N) vote_groups
```

**说明**：
- 每个年度可配置多条考核规则（按周期、对象类型、对象分类）
- 每条规则包含多个分数模块（直接录入、投票、加减分）
- 投票模块包含多个投票分组

### 4.6 分数数据关系

```
assessment_years (1) ----< (N) direct_scores
score_modules (1) ----< (N) direct_scores
assessment_objects (1) ----< (N) direct_scores

assessment_years (1) ----< (N) vote_tasks
vote_groups (1) ----< (N) vote_tasks
assessment_objects (1) ----< (N) vote_tasks
employees (1) ----< (N) vote_tasks
vote_tasks (1) ----< (N) vote_records

assessment_years (1) ----< (N) extra_points
score_modules (1) ----< (N) extra_points
assessment_objects (1) ----< (N) extra_points
```

**说明**：
- 直接录入分数关联年度、模块、考核对象
- 投票任务关联年度、投票分组、考核对象、投票人
- 每个投票任务对应一条投票记录
- 加减分关联年度、模块、考核对象

### 4.7 计算结果关系

```
assessment_years (1) ----< (N) calculated_scores
assessment_objects (1) ----< (N) calculated_scores
assessment_rules (1) ----< (N) calculated_scores

assessment_years (1) ----< (N) rankings
assessment_objects (1) ----< (N) rankings
```

**说明**：
- 计算结果关联年度、考核对象、使用的规则
- 排名关联年度、考核对象

### 4.8 级联删除策略

| 父表 | 子表 | 级联策略 | 说明 |
|------|------|----------|------|
| assessment_years | assessment_periods | CASCADE | 删除年度时删除所有周期 |
| assessment_years | assessment_objects | CASCADE | 删除年度时删除所有考核对象 |
| assessment_years | assessment_rules | CASCADE | 删除年度时删除所有规则 |
| assessment_years | direct_scores | CASCADE | 删除年度时删除所有分数数据 |
| assessment_years | vote_tasks | CASCADE | 删除年度时删除所有投票任务 |
| assessment_years | extra_points | CASCADE | 删除年度时删除所有加减分 |
| assessment_years | calculated_scores | CASCADE | 删除年度时删除所有计算结果 |
| assessment_years | rankings | CASCADE | 删除年度时删除所有排名 |
| assessment_rules | score_modules | CASCADE | 删除规则时删除所有模块 |
| score_modules | vote_groups | CASCADE | 删除模块时删除所有投票分组 |
| vote_tasks | vote_records | CASCADE | 删除任务时删除投票记录 |
| users | user_roles | CASCADE | 删除用户时删除角色关联 |
| roles | user_roles | CASCADE | 删除角色时删除用户关联 |

**说明**：
- 年度数据采用级联删除，确保数据完整清理
- 其他关键数据（组织、部门、员工）采用软删除，不使用级联删除

### 6.2 核心业务流程关系

```
年度创建流程：
assessment_years → assessment_periods (自动创建5个周期)

考核对象配置流程：
assessment_years → assessment_objects (关联 organizations/departments/employees)

规则配置流程：
assessment_years → assessment_rules → score_modules → vote_groups

分数录入流程：
assessment_objects + score_modules → direct_scores
assessment_objects + vote_groups → vote_tasks → vote_records
assessment_objects + score_modules → extra_points

分数计算流程：
direct_scores + vote_records + extra_points → calculated_scores → rankings
```

## 附录

### A. 数据精度说明

1. **分数字段**: DECIMAL(10, 6)
   - 整数部分: 4位（最大9999.999999）
   - 小数部分: 6位（保证计算精度）
   - 显示时根据系统配置四舍五入到指定位数

2. **权重字段**: DECIMAL(5, 4)
   - 整数部分: 1位（0-1之间）
   - 小数部分: 4位（如0.7000）
   - 权重之和必须等于1.0000

3. **时间戳**: INTEGER
   - Unix timestamp（秒级）
   - 使用 strftime('%s', 'now') 获取当前时间戳

### B. 性能优化建议

1. **定期维护**:
   ```sql
   -- 分析表统计信息
   ANALYZE;

   -- 清理碎片
   VACUUM;

   -- 重建索引
   REINDEX;
   ```

2. **查询优化**:
   - 使用索引字段作为查询条件
   - 避免在 WHERE 子句中对索引字段使用函数
   - 使用 EXPLAIN QUERY PLAN 分析查询计划

3. **数据归档**:
   - 建议每年归档历史年度数据
   - 保留最近3年的在线数据
   - 归档数据可导出为独立数据库文件

### C. 安全建议

1. **密码安全**:
   - 使用 bcrypt 加密，cost=10
   - 强制首次登录修改密码
   - 定期提醒用户修改密码

2. **数据备份**:
   - 启用自动备份功能
   - 定期测试备份恢复
   - 异地保存重要备份

3. **审计日志**:
   - 记录所有关键操作
   - 定期审查异常操作
   - 日志保留至少1年
