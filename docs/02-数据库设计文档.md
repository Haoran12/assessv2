# AssessV2 数据库设计文档（当前实现）

- 版本：v3.2
- 日期：2026-03-24
- 迁移基线：`business 0001~0010`、`accounts 1001~1002`

## 1. 数据库总体

系统采用双库：

- 业务库：`data/assess.db`
- 账号库：`data/accounts/accounts.db`

## 2. 业务库核心表

## 2.1 `assessment_sessions`

| 列名 | 类型 | 说明 |
| --- | --- | --- |
| id | INTEGER PK | 场次 ID |
| assessment_name | VARCHAR(200) UNIQUE | 场次目录名 |
| display_name | VARCHAR(200) | 场次展示名 |
| year | INTEGER | 年份 |
| organization_id | INTEGER | 所属组织 |
| status | VARCHAR(20) | `preparing/active/completed` |
| completed_snapshot_path | VARCHAR(500) | 完成快照路径 |
| completed_snapshot_created_at | INTEGER | 完成快照创建时间 |
| description | TEXT | 描述 |
| data_dir | VARCHAR(500) | 场次数据目录 |
| created_by | INTEGER | 创建人 |
| created_at | INTEGER | 创建时间戳 |
| updated_by | INTEGER | 更新人 |
| updated_at | INTEGER | 更新时间戳 |

约束说明：

- `assessment_name` 唯一。
- 未设置 `(year, organization_id)` 唯一约束。

## 2.2 `assessment_session_periods`

- 用途：场次周期定义（含 `rule_binding_key`）。
- 关键约束：`UNIQUE(assessment_id, period_code)`。

## 2.3 `assessment_object_groups`

- 用途：场次对象分组。
- 关键约束：`UNIQUE(assessment_id, group_code)`。

## 2.4 `assessment_session_objects`

- 用途：场次对象快照。
- 关键约束：`UNIQUE(assessment_id, target_type, target_id)`。

## 2.5 `assessment_object_module_scores`

- 用途：对象维度模块评分写入。
- 与场次状态联动：`completed` 场次禁止写入。

## 2.6 `rule_files`

- 用途：规则文件存储与权限控制。
- 关键字段：`assessment_id`、`content_json`、`file_path`、`is_copy`、`source_rule_id`、`owner_org_id`。

## 2.7 `assessment_rule_bindings`

- 用途：规则绑定关系。
- 关键约束：`UNIQUE(assessment_id, period_code, object_group_code, organization_id)`。

## 2.8 备份相关表

- `backups`：全量备份与组织逻辑备份记录。
- 当前支持 `full_snapshot` 与组织包相关字段（见迁移 `0007`）。

## 2.9 审计相关表

- `audit_logs`：统一审计记录（结构升级见迁移 `0008`）。

## 3. 账号库核心表

- `users`
- `roles`
- `user_roles`
- `user_organizations`
- `user_permission_bindings`
- `audit_logs`

## 4. 与场次状态联动的数据库规则

1. 创建场次默认状态：`preparing`。
2. 切换到 `completed` 后写入：
- `completed_snapshot_path`
- `completed_snapshot_created_at`
3. 应用层对 `completed` 状态执行统一写保护。

## 5. 快速核查 SQL

```sql
-- 场次状态字段
PRAGMA table_info('assessment_sessions');

-- 状态索引
SELECT name FROM sqlite_master
WHERE type='index' AND name='idx_assessment_sessions_status';

-- 场次状态分布
SELECT status, COUNT(*) AS cnt
FROM assessment_sessions
GROUP BY status;
```
