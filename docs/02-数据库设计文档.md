# AssessV2 数据库设计文档（字段字典与 API 映射版）

- 版本：v3.1
- 日期：2026-03-18
- 口径：`business 0001 + 0002`、`accounts 1001 + 1002`

## 1. 数据库总体

系统采用双库部署：

- 业务库：`./data/assess.db`
- 账号库：`./data/accounts/accounts.db`

业务库承载组织、场次、规则、备份、审计等。
账号库承载用户、角色、组织授权与账号审计。

## 4. API 字段级映射表（对接重点）

## 4.1 创建场次 API 映射

接口：`POST /api/assessment/sessions`

| 请求字段 | 类型 | 入库列 | 说明 |
| --- | --- | --- | --- |
| year | number | `assessment_sessions.year` | 年份 |
| organizationId | number | `assessment_sessions.organization_id` | 归属组织 |
| displayName | string | `assessment_sessions.display_name` | 为空时后端生成 |
| description | string | `assessment_sessions.description` | 描述 |

自动生成入库：

| 数据对象 | 入库表 | 规则 |
| --- | --- | --- |
| 默认周期 | `assessment_session_periods` | 5 条默认周期 |
| 默认对象分组 | `assessment_object_groups` | 7 条默认分组 |
| 默认对象 | `assessment_session_objects` | 基于组织数据生成 |

## 4.2 替换周期 API 映射

接口：`PUT /api/assessment/sessions/:id/periods`

| 请求字段 | 类型 | 入库列 | 说明 |
| --- | --- | --- | --- |
| items[].periodCode | string | `period_code` | 后端统一大写 |
| items[].periodName | string | `period_name` | 周期显示名 |
| items[].sortOrder | number | `sort_order` | 后端按数组顺序重排 |

## 4.3 替换对象分组 API 映射

接口：`PUT /api/assessment/sessions/:id/object-groups`

| 请求字段 | 类型 | 入库列 | 说明 |
| --- | --- | --- | --- |
| items[].objectType | string | `object_type` | `team/individual` |
| items[].groupCode | string | `group_code` | 场次内唯一 |
| items[].groupName | string | `group_name` | 展示名称 |
| items[].sortOrder | number | `sort_order` | 后端按顺序重排 |

## 4.4 替换对象 API 映射

接口：`PUT /api/assessment/sessions/:id/objects`

| 请求字段 | 类型 | 入库列 | 说明 |
| --- | --- | --- | --- |
| items[].objectType | string | `object_type` | 对象类型 |
| items[].groupCode | string | `group_code` | 分组编码 |
| items[].targetType | string | `target_type` | 源实体类型 |
| items[].targetId | number | `target_id` | 源实体 ID |
| items[].sortOrder | number | `sort_order` | 排序 |
| items[].isActive | boolean | `is_active` | 是否启用 |

父子关系处理：

- 先按 target 建立对象记录
- 再回填 `parent_object_id`

## 4.5 规则创建 API 映射

接口：`POST /api/rules/files`

| 请求字段 | 类型 | 入库列 | 说明 |
| --- | --- | --- | --- |
| assessmentId | number | `rule_files.assessment_id` | 场次 ID |
| ruleName | string | `rule_files.rule_name` | 规则名 |
| description | string | `rule_files.description` | 描述 |
| contentJson | string | `rule_files.content_json` | 规则 JSON |

自动写入：

| 结果 | 入库列 |
| --- | --- |
| 物理文件路径 | `rule_files.file_path` |
| 是否拷贝 | `rule_files.is_copy=false` |

## 4.6 规则绑定 API 映射（拷贝）

接口：`POST /api/rules/bindings/select`

第一阶段（创建拷贝）：

| 来源 | 入库表/列 | 说明 |
| --- | --- | --- |
| sourceRuleId | `rule_files.source_rule_id` | 记录来源规则 |
| 目标组织 | `rule_files.owner_org_id` | 拷贝归属组织 |
| 源规则内容 | `rule_files.content_json` | 内容复制 |
| 文件生成 | `rule_files.file_path` | 新文件路径 |
| 拷贝标记 | `rule_files.is_copy=true` | 拷贝规则 |

第二阶段（绑定关系）：

| 请求字段 | 入库列 |
| --- | --- |
| assessmentId | `assessment_rule_bindings.assessment_id` |
| periodCode | `assessment_rule_bindings.period_code` |
| objectGroupCode | `assessment_rule_bindings.object_group_code` |
| 组织上下文 | `assessment_rule_bindings.organization_id` |
| 新拷贝规则 ID | `assessment_rule_bindings.rule_file_id` |

## 5. 旧结构移除说明

业务迁移 `0002` 已移除旧主流程表（例如 `assessment_years`、`assessment_rules`、`vote_tasks` 等）。
这些名称仅可用于历史核查，不应再作为新对接目标。

## 6. 快速核查 SQL

```sql
-- 核查新核心表
SELECT name FROM sqlite_master
WHERE type='table'
  AND name IN (
    'assessment_sessions',
    'assessment_session_periods',
    'assessment_object_groups',
    'assessment_session_objects',
    'rule_files',
    'rule_file_hides',
    'assessment_rule_bindings'
  );

-- 核查旧年度主表是否移除
SELECT name FROM sqlite_master
WHERE type='table' AND name='assessment_years';
```
